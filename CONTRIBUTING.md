# Contributing to E-Commerce Management System

First off, thank you for considering contributing to the E-Commerce Management System! It's people like you that make this tool better for everyone.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

- **Check the existing issues** to see if the bug has already been reported.
- **Provide as much detail as possible** in your issue report, including steps to reproduce, expected behavior, and actual behavior.

### Suggesting Enhancements

- **Open a new issue** describing the enhancement you'd like to see.
- **Explain why this enhancement would be useful** and how it fits into the project's goals.

### Pull Requests

- **Follow the existing code style**: We use Ruff for linting and Black for formatting.
- **Ensure all tests pass**: Run `pytest` before submitting your PR.
- **Keep PRs focused**: Try to address only one issue or feature per PR.
- **Write meaningful commit messages**: Follow the project's commit message convention (if any).

## Development Setup

See the [Getting Started](README.md#getting-started) section in the README for instructions on setting up your development environment.

## Architecture

Please familiarize yourself with our [Architecture Pattern](README.md#architecture-pattern) and the [Service Layer Pattern](docs/01-ARCHITECTURE.md) before making changes.

## Questions?

Feel free to open an issue for any questions you might have!
