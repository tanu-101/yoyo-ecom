# Plan Feature Implementation

Before implementing anything, create a detailed plan following this structure:

## Context
What is the feature request? What problem does it solve?

## Analysis
1. Which apps does this affect?
2. What models need to be created/modified?
3. What services need to be created/modified?
4. What API endpoints are needed?
5. What tests need to be written?

## Implementation Plan
Step-by-step plan:
1. [ ]
2. [ ]
3. [ ]

## Files to Create/Modify
- `apps/<name>/models/...`
- `apps/<name>/services/...`
- `apps/<name>/selectors/...`
- `apps/<name>/views/...`
- `apps/<name>/serializers/...`
- `apps/<name>/tests/...`

## Verification
- [ ] Run tests
- [ ] Run linting
- [ ] Test manually

---

After creating the plan, wait for approval before implementing.
